# portals-pg-archive

## 项目简介

`portals-pg-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为页面资源的结构化存储与展示空间。  
所有页面内容均为静态 HTML，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

```
portals-pg-archive/
├── index.html          # 入口页面，提供归档索引
├── pages/              # 存放各独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的主入口，通常包含页面列表或导航链接。
- `pages/`：所有归档的 HTML 页面存放于此，文件名建议保持有意义的命名。
- `assets/`：存放页面共用的样式、脚本或图片资源，便于统一维护。

## 页面归档说明

- 每个页面均为独立、完整的 HTML 文件，不依赖外部数据库或服务端环境。
- 页面内容通常来自过往项目、临时页面或需要长期保留的静态快照。
- 归档时请尽量保留原始页面结构与样式，若需修改建议在文件头部添加注释说明。
- 本仓库不提供页面内容的实时更新，所有归档文件均为历史状态。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- 更新资源：修改 `assets/` 中的公共文件时，注意检查各页面引用是否正常。
- 删除页面：移除对应文件并更新索引，不建议频繁删除已归档内容。
- 本仓库欢迎提交 Pull Request，但请确保新增内容不包含敏感信息或违反平台规定。
